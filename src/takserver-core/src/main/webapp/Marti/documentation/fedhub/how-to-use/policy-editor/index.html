Policy Editor

Once you have created a new policy or loaded an existing one, you will be brought to the graph editor. This is where you will manage how the federates connecting to the Fedhub are allowed to interact with each other.

Add / Modify CA Group Node​

A Certificate Authority (CA) group represents the CA of a TAK Server/Fedhub that wants to connect to this Fedhub. To add a CA Group to the graph simply drag a "CA Group" Node onto the canvas. You will be presented with a form. You will want to browse the list of available CA's by either their issuer or subject name and select the one you want to configure.

You will see two checkboxes at the bottom.

  • Enabling "Interconnected" means that if multiple servers are connected to the hub and they are using the same CA, they will be automatically linked together and therefore share all of their data with each other. (Enabling this is very uncommon as it is recomended to run one set of unique certs per server)
  • Allow Token Auth When enabled, will automatically allow incoming federate connections to present thier client certificates over TLS (instead of traditional mTLS) to authenticate and receive a token. This will allow federation connections to work in network environments where mutual authentication is not an option (Ex. if your network is running break and inspect on all tls connections). The Token Auth Duration is how often the clients token will expire forcing them to reconnection. A value of -1 means the token will not expire. Any value greater than 0 may cause disruptions in service until the remote end attemps to reconnect.

Add / Modify Outgoing Node​

An Outgoing Connection node will establish a connection from this Fedhub to another TAK Server or Fedhub. To add an Outgoing Connection to the graph simply drag a "Outgoing" Node onto the canvas. You will be presented with a form. Enter the connection information of the other server and select the enabled checkbox when you are ready for the connection to be initiated.

Add / Modify Token Group Node​

A Token group represents a set of token(s) that TAK Servers/Fedhubs can use to connect to this Fedhub. To add a Token Group to the graph simply drag a "Token Group" Node onto the canvas. You will be presented with a form. Here you can generate and manage your active tokens. A value of -1 means the token will not expire. The difference between a token group and the CA Group token auth mentioned above is that the CA Group is automatically given a token by presenting it's cert. In this case for a Token Group, it is more of a manual process as you will need to send the other server's admin a new token when theirs expires.

Connections Groups​

Next we will connect these CA groups / token groups / outgoing connections to each other with "edges" to begin federating data. Edges are unidirectional so to do this, drag from the left or right of the source node and complete the drag on the destination node. The edge arrow will indicate the flow of traffic. When the edge between nodes is created, a form will appear and ask you how you want to configure group filtering.

Filtering options:​

  • Allow All Messages This must be used for all V1 server connections and any V2 server connections that aren't running group mapping. This will allow all messages without performing any filtering
  • Allowed Groups Allow any message that contains at least one of these groups
  • Disallowed Groups Block any message containing one of these groups
  • Allowed And Disallowed Groups Allow messages that contains an allowed group and contains no disallowed groups

Note: The source node must be running group mapping to use a filtering option other than Allow All Messages

Update a Policy​

Each time you add a node to the graph, delete a node, edit a node, add a connection, generate tokens or make any other changes on the graph you will need to save and upload the policy for it to 1. Save your changes and 2. Take effect. To save the policy and set the updates as active, click the cloud icon on the top toolbar. Once the policy is active, federates can connect to the Hub. The Federation Hub will deny connections from servers that are not present in the Hub's policy.
