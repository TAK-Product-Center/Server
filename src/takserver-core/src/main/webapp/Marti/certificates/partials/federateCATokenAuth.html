When enabled, will allow incoming federate connections to present thier client certificates over TLS (instead of traditional mTLS) to authenticate and receive a token. This will allow federation connections to work in network environments where mutual authentication is not an option (Ex. if your network is running break and inspect on all tls connections).

Note: For this to work you will also need to enable token authentication and set an additional port in the federation config settings
